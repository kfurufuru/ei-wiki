## 6. 改修計画

### Phase 0: 緊急止血 — 安全直結誤りの修正（即日〜3日、S級14件）

- 完了基準: S級一覧が全て解消し、接地・絶縁・耐圧の数値がページ間で一致

### Phase 1: 数値の正典化（1〜2週）

- 「正典ページ」を指定し、他ページは要約＋リンクに改める:
    - 接地種別・抵抗値 → `guidelines/grounding-inspection.md`（現状最も正確）
    - 絶縁抵抗判定・メガー印加電圧 → `05-hozen/insulation-management.md`（電技58条法定値を追加のうえ）
    - 耐圧試験電圧 → `01-koatsu/insulation-test.md`

- `includes/` にスニペット化（pymdownx.snippets導入済み）して数値表を単一ソース化する方式も検討
- A級18件を並行して修正
- 完了基準: grepで「D種」「メガー」「耐圧」を引いたとき、数値が全ページで単一の正典に一致

### Phase 2: 構造改修（2〜4週）

- K1: `trouble-first-response.md` を 06-trouble へ移設し初動の一本化。basics=概念、abnormal=非定常特化に役割再定義。リダイレクト（mkdocs-redirectsプラグイン）でURL維持

### Phase 4: 運用強化 — 再発防止（Phase 1と並行着手可）

- CI強化（`pr-check.yml`に追加）:
    - frontmatter必須キー（audience含む）欠落でFAIL
    - 06-troubleの `search: boost` 欠落でFAIL
    - コードブロック言語未指定をWARN表示
    - **数値矛盾lint**: 正典化した数値（D種100Ω等）の再定義をgrepで検出しFAIL
- 鮮度監査の実効化: last_verified欠落をFAILに格上げ。一括日付の解消は「実際に内容検証したページのみ日付更新」を原則化（日付の機械的書き換えは禁止）
- `11-genai/` のモデル固有名を抽象表現へ置換し陳腐化を止める
- 完了基準: 本計画のS/A級と同種の誤りがCIで機械検出される状態

### 実施順序まとめ

```
Phase 0（即日） → Phase 1 + Phase 4（並行、〜2週） → Phase 2（〜1ヶ月） → Phase 3（〜2ヶ月）
```

数値誤り修正（Phase 0/1）を新規コンテンツ（Phase 3）より必ず先行させる。**誤った数値を掲げたまま網羅性を上げるのは、信頼できない参照先を増やすだけ**のため。
