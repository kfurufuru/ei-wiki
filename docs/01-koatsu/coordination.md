# 保護協調

## 30秒まとめ

保護協調の目的は「事故点に最も近い遮断器だけが開放し、停電範囲を最小にすること」。TCC（時間-電流特性曲線）で上位の曲線が下位の曲線より常に上（＝遅い）にあり、時限差 0.3〜0.4 秒以上が保たれることを確認する。整定はタップ（電流整定）とレバー（時間整定 TMS）の2つで決める。変圧器増設・励磁突入・コンデンサ投入で協調が崩れやすいので、変更時は必ず TCC を引き直す。

---

## 保護協調の目的

化学プラントでは停電範囲の最小化が連続生産維持の観点から非常に重要です。

```
理想の動作：事故点→最近傍遮断器（最下位）のみトリップ
避けるべき動作：上位の遮断器がトリップして広範囲停電
```

### 保護協調が成立している状態

```
上位（母線側） [時限 0.8秒] ─┐
                              │
下位（フィーダー） [時限 0.4秒] ─┤ 差：0.4秒 → 協調成立
                              │
最下位（負荷直近） [時限 0.1秒] ─┘ 差：0.3秒 → 協調成立
```

---

## TCC（時間-電流特性曲線）の読み方

TCC（Time-Current Curve）は、横軸に電流、縦軸に動作時間をとった両対数（対数-対数）グラフです。横軸は整定電流（ピックアップ電流）に対する倍数 M で読むと機器間の比較がしやすくなります。

### グラフ上で何を見るか

- **縦方向の位置関係**: ある電流値に垂直線を立てたとき、上位機器の曲線が下位機器の曲線より **上（動作時間が長い）** にあれば、その電流では下位が先に動作します。
- **重なり・交差の禁止**: 想定する事故電流の全範囲（下は限時ピックアップ、上は最大短絡電流）で、上位曲線が下位曲線に接触・交差しないことが協調の条件です。
- **時限差の確保**: 同一電流での上位・下位の動作時間の差（縦の間隔）が、下記の時限差以上あることを確認します。

```
時限差 = 下位機器の動作時間 + VCB 動作時間（約0.05秒）+ 協調余裕（0.1〜0.2秒）
       ≒ 0.3〜0.4秒
```

!!! note "両対数グラフである理由"
    電流は定格の数倍〜数十倍、時間は 0.01 秒〜数十秒と、いずれも桁で変化します。両対数にすると反限時特性が概ね直線状になり、複数曲線の上下関係を目視で判定しやすくなります。

### 反限時特性（IDMT）の種類と使い分け

反限時（IDMT: Inverse Definite Minimum Time）特性は「電流が大きいほど速く動作する」曲線です。IEC 60255-151 は動作時間を次式で定義します（ユニコード表記）。

```
t = ( K / (M^E − 1) ) × TMS

t   ：動作時間 [秒]
M   ：電流倍数（= 事故電流 / 整定電流。M > 1 で動作）
TMS ：時間整定（Time Multiplier Setting、レバー）。曲線を時間方向に上下させる係数
K, E：特性ごとの定数（下表）
```

| 特性 | 略号 | K | E | 傾きの特徴 | 主な使いどころ |
|------|------|-----|------|-----------|--------------|
| 標準反限時 | SI（Standard Inverse） | 0.14 | 0.02 | ゆるやか | 一般フィーダ・汎用。電流変化に対し時間が緩慢 |
| 超反限時 | VI（Very Inverse） | 13.5 | 1 | 中程度 | 電源側インピーダンスが小さく、遠近で短絡電流差が大きい系統 |
| 超々反限時 | EI（Extremely Inverse） | 80 | 2 | 急峻 | 高調波・突入の多い変圧器/モーター、ヒューズとの協調 |
| 長限時 | LTI（Long Time Inverse） | 120 | 1 | 非常にゆるやか | 過負荷保護主体（熱的保護） |

同じ TMS = 1.0 で各特性の動作時間を倍数ごとに並べると、傾きの違いが読み取れます（式に代入した計算値）。

| 電流倍数 M | SI [秒] | VI [秒] | EI [秒] |
|-----------|--------|--------|--------|
| 2 倍 | 10.0 | 13.5 | 26.7 |
| 5 倍 | 4.28 | 3.38 | 3.33 |
| 10 倍 | 2.98 | 1.50 | 0.81 |
| 20 倍 | 2.27 | 0.71 | 0.20 |

!!! tip "特性選定の勘どころ"
    大電流域で下位を素早く切り、小電流域では鈍く待たせたい（＝下位ヒューズや変圧器突入との協調をとりたい）ほど EI 寄りを選びます。上位と下位で特性を揃えると曲線が平行になり、全電流域で一定の時限差を保ちやすくなります。異なる特性を混在させると、ある電流域で時限差が詰まって交差することがあるため、TCC 全域での確認が必須です。

## 整定の実例（6.6kV フィーダと上位母線の協調）

下位フィーダ OCR と上位母線 OCR を、タップ（電流整定）×レバー（TMS）でどう決めるかを 1 例で示します。数値は [保護継電器](relay.md) の整定計算例、および [短絡電流計算](../04-sekkei/fault-current.md) の %Z 法と整合させています。

### 前提

- 系統: 6.6kV、1000 kVA 変圧器フィーダが母線から分岐。
- 短絡電流: 母線 3 相短絡で **約 1094 A**（%Z 合計 8 % による計算値。導出は [短絡電流計算](../04-sekkei/fault-current.md) を参照。ここでは結果のみ引用）。
- 下位フィーダの変圧器定格電流 = 1000 kVA / (√3 × 6.6 kV) = **87.5 A**（[保護継電器](relay.md) と同一）。

### 手順 1: 下位フィーダ OCR のタップ選定

負荷（変圧器定格）電流を基準に、過負荷で不要動作しない値へ設定します。

```
CT 変流比：100/5 A（変流比 20）
CT 二次換算の定格電流 = 87.5 / 20 = 4.375 A
限時タップ = 4.375 × 1.2 ≒ 5.25 A → 5.5 A タップに設定
（一次換算 = 5.5 × 20 = 110 A。定格の約 1.26 倍）
```

### 手順 2: 下位の短絡時動作時間を読む

短絡電流 1094 A における下位 OCR の倍数と、SI 特性・TMS = 0.1 での動作時間を求めます。

```
倍数 M = 短絡電流 / 一次整定電流 = 1094 / 110 ≒ 9.9 倍
t = (0.14 / (9.9^0.02 − 1)) × 0.10 ≒ 2.98 × 0.10 ≒ 0.30 秒
```

下位フィーダは短絡に対し **約 0.30 秒** で動作します。

### 手順 3: 上位母線 OCR のレバー（TMS）決定

上位は「下位の動作時間 + 時限差 0.3〜0.4 秒」を狙って TMS を逆算します。母線には複数フィーダが集まるため、タップは合計負荷に見合う高めの値とします。

```
上位 CT：400/5 A、限時タップ 3.75 A（一次換算 300 A）
上位が見る短絡時倍数 M = 1094 / 300 ≒ 3.6 倍
目標動作時間 = 下位 0.30 秒 + 時限差 0.34 秒 ≒ 0.64 秒

SI 特性で t = 0.64 秒となる TMS を逆算：
0.64 = (0.14 / (3.6^0.02 − 1)) × TMS = 5.34 × TMS
TMS ≒ 0.64 / 5.34 ≒ 0.12
```

### 整定結果のまとめ

| 階層 | CT | 限時タップ（一次換算） | 特性 | レバー TMS | 短絡時倍数 M | 動作時間 |
|------|-----|-------------------|------|-----------|-----------|---------|
| 下位（フィーダ） | 100/5 A | 5.5 A（110 A） | SI | 0.10 | 約 9.9 倍 | 約 0.30 秒 |
| 上位（母線） | 400/5 A | 3.75 A（300 A） | SI | 0.12 | 約 3.6 倍 | 約 0.64 秒 |

短絡電流 1094 A において **時限差 ≒ 0.34 秒** が確保され、上位曲線が下位曲線の上に余裕をもって乗るため協調が成立します。

!!! note "なぜ上位は倍数が小さいのに遅くできるか"
    上位はタップ（整定電流）が高いぶん、同じ短絡電流でも倍数 M が小さくなります。反限時特性では M が小さいほど動作が遅いため、上位は下位より自然に遅くなります。さらにレバー（TMS）で微調整して所要の時限差に合わせます。逆に上位下位のタップが近いと倍数差がつかず、レバーだけで時限差を作ることになるため、TCC 全域で交差しないか要確認です。

### 瞬時要素（高速遮断）の扱い

限時要素だけでは大電流時の動作が遅い場合、瞬時要素を併用します。整定の考え方は [保護継電器](relay.md)・[変圧器](transformer.md) の正典に従います。

- 下位フィーダの瞬時整定は **定格の 6〜10 倍（600〜1000 %）** 以上。フィーダ先変圧器の励磁突入電流（定格の 6〜10 倍）を上にまたぐ値とし、突入での不要トリップを避けます。
- 瞬時要素は時限協調の対象外（ほぼ 0 秒動作）なので、上位の瞬時整定は下位の短絡電流域に踏み込まないよう「上位から見た背後（母線）短絡のみ」を検出する値に離します。

## 上位-下位の時限協調

| 時限差の目安 | 条件 |
|-----------|------|
| 0.3秒 | VCB + OCR の最低限の時限差 |
| 0.4秒 | 余裕を持たせた標準的な時限差 |
| 0.5秒以上 | 大型系統・古い機器での推奨 |

時限階層の整定目安（最下位 0.1〜0.3 秒 / 中間 0.4〜0.7 秒 / 上位 0.8〜1.2 秒）は [保護継電器](relay.md) を正典とします。

## 協調が崩れる典型と確認方法

協調は一度組めば終わりではなく、系統や機器の変更で崩れます。代表的な崩れ方と、TCC 上での確認ポイントを示します。

### 変圧器の励磁突入電流と瞬時要素

変圧器投入時、定格の **6〜10 倍** の突入電流が数サイクル〜0.1 秒程度流れます（正典: [変圧器](transformer.md)）。

- 瞬時要素をこの突入より下に置くと、投入のたびに不要トリップします。瞬時整定は突入電流の上をまたぐ値（600〜1000 % 以上）にします。
- TCC 上では、突入電流の点（横軸 = 定格の 6〜10 倍、縦軸 = 継続時間 0.1 秒程度）が OCR 曲線より下（左下）に収まっているかを確認します。
- 第2高調波抑制機能付き OCR は突入電流に多く含まれる第2高調波を判別して動作を抑制するため、より低い瞬時整定でも突入をまたげます。

### 遮断容量の超過

短絡電流が既設遮断器の定格遮断電流を超えると、そもそも遮断できず協調以前の問題になります。

- 変圧器増設・並列で短絡電流が増えたら、まず各遮断器の定格遮断電流 ≥ 想定短絡電流を確認します（計算法は [短絡電流計算](../04-sekkei/fault-current.md)）。
- 遮断容量が不足する場合は、遮断器更新・限流ヒューズ追加・系統分割（母線タイ開放運用）で短絡電流を抑えます。

### 進相コンデンサの投入突入

コンデンサ投入時にも高周波・大振幅の突入電流が流れます。特に複数バンクの後段投入（バンク間充放電）では突入が大きくなります。

- コンデンサ回路の OCR・ヒューズが投入突入で動作しないよう、直列リアクトル（一般に 6 % 品）で突入を抑制し、TCC 上で突入点が曲線より下にあることを確認します。
- 力率改善制御で頻繁に投入・開放する場合は、突入の繰り返しに対する耐量も併せて確認します。

### 変更時の TCC 再確認（チェックの型）

!!! warning "変圧器増設・系統変更時の再確認手順"
    | ステップ | 内容 |
    |---------|------|
    | 1 | 変更後の短絡電流を再計算（[短絡電流計算](../04-sekkei/fault-current.md)） |
    | 2 | 各遮断器の定格遮断電流 ≥ 変更後短絡電流を確認 |
    | 3 | 新旧の負荷電流に合わせて下位タップを見直し |
    | 4 | TCC を引き直し、全電流域で上位曲線が下位曲線の上（時限差 0.3 秒以上）にあるか確認 |
    | 5 | 励磁突入・コンデンサ投入突入の点が曲線の下に収まるか確認 |
    | 6 | 整定変更したら注入試験で実動作を確認（[保護リレー試験手順](../guidelines/protective-relay-test.md)） |

## 作成ツールの現実 — 丸投げにしない

TCC の作図・協調検討は、メーカー（三菱電機・富士電機・東芝等）の保護協調検討ソフトに機器データを入力して行うのが一般的です。ただしソフトはあくまで作図・判定の道具であり、入力値と結果の妥当性はエンジニアが判断する必要があります。

!!! tip "エンジニアが最低限おさえること"
    - **入力の正しさ**: CT 変流比・整定タップ・特性種別・TMS・短絡電流を、銘板と系統計算から正しく入れる。ゴミを入れれば協調 OK でも実系統では崩れます。
    - **結果の検算**: 代表 1 点（例: 最大短絡電流での下位動作時間）を上式 `t = (K/(M^E−1))×TMS` で手計算し、ソフト出力と一致するか確かめる。本ページの整定例がその検算の型です。
    - **崩れの当たり付け**: 「増設で短絡が増えた」「特性を混在させた」「突入をまたげていない」など、崩れる典型を知っていれば、ソフトが出す曲線の異常に気づけます。
    - 外注する場合も、上記が読めれば受領した協調図をレビューでき、「メーカー任せで中身が分からない」状態を避けられます。

## 保護協調図（TCC）の保管

保護協調図は以下の変更が発生したときに必ず更新します。

- [ ] 変圧器の増設・変更
- [ ] 高圧モーターの増設・変更
- [ ] 系統構成変更（母線分割等）
- [ ] 保護継電器の更新・整定変更
- [ ] 進相コンデンサの増設
