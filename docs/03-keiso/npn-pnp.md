# NPN/PNP 選定

## 30秒まとめ

センサの出力タイプ（NPN/PNP）とPLCのDI入力タイプ（シンキング/ソーシング）の**組み合わせが合わないと接続できない**。日本製センサはNPN出力が多く、欧州製はPNP出力が多い。既設PLCのDI仕様を先に確認してからセンサを選定するのが原則。混在する場合はリレーかフォトカプラで変換する。

---

## 基本概念

NPN・PNPはトランジスタの型名であり、センサの**出力信号の電流方向**を表す。

### NPN出力（シンク型）

信号ONのとき、出力端子からGNDへ電流を**引き込む（シンク）**。

```
+24V ──────── 負荷（PLCのDI入力）── 出力端子 ─[NPN Tr]─ GND
                                              ON時：電流がGNDへ流れる
```

- 出力端子はON時に「LOW（GND電位）」になる
- **日本製センサに多い**（国内標準）
- PLC側は「シンキング（ソース型）入力」と接続する

### PNP出力（ソース型）

信号ONのとき、出力端子からDI入力へ電流を**供給する（ソース）**。

```
+24V ─[PNP Tr]─ 出力端子 ── 負荷（PLCのDI入力）──── GND
                       ON時：電流が+24Vから流れ出す
```

- 出力端子はON時に「HIGH（+24V電位）」になる
- **欧州製センサに多い**（NAMUR等欧州規格準拠機器）
- PLC側は「ソーシング（シンク型）入力」と接続する

---

## PLC DI入力タイプとの対応表

| センサ出力 | 電流方向 | 対応するPLC DI入力 | 国内の多数派 |
|-----------|---------|------------------|-----------|
| NPN（シンク型） | 負荷→センサ→GND | ソース型入力（+COM） | ✅ 多い |
| PNP（ソース型） | +24V→センサ→負荷 | シンク型入力（-COM） | ─ 少ない |

!!! warning "用語の混乱に注意"
    「シンク型センサ = ソース型入力に接続する」という**センサとPLC入力で呼称が逆転する**。
    メーカーカタログの表現がまちまちなため、配線図で電流の流れを確認するのが確実。

## 選定フロー

=== "既設PLC"

    ```
    ① PLCのDI入力ユニット仕様書を確認
          ↓
       ソース型入力（+COM）？ → NPN出力センサを選定
       シンク型入力（-COM）？ → PNP出力センサを選定
       NPN/PNP両対応？      → NPN優先で選定
          ↓
    ② センサカタログで出力タイプ（NPN/PNP）を確認
          ↓
    ③ 混在が発生する場合 → リレー or フォトカプラで変換
    ```

=== "新設PLC（推奨手順）"

    ```
    ① PLCのDI入力ユニットを「NPN/PNP両対応タイプ」で指定する
          ↓
    ② センサはNPN出力で統一する
          ↓
    ③ 将来PNP機器が混入してもCOM配線変更のみで対応可能
    ```

    !!! tip "仕様書への明記"
        発注仕様書に「DI入力：NPN/PNP両対応、DC24V、16点」と明記する。
        「汎用DI」だけだと片方専用品が納入されることがある。

## 新設PLCでNPNを選ぶ理由

新設PLCであれば入力タイプを自由に決められるが、それでも**NPN統一を推奨する**理由は以下の5点。

| 理由 | 内容 |
|------|------|
| 国内PLCのデフォルト | 三菱・オムロン・富士電機等はソース型（+COM）がデフォルト。追加設定不要 |
| 調達性 | 国内市場の近接・光電センサはNPN標準品が主流。緊急調達しやすい |
| 既設設備との統一 | 増設・改造時に混在が起きない |
| 長距離配線の安定性 | PNPより漏れ電流による誤ON検知が起きにくい |
| 安全バリアとの親和性 | 本安防爆用バリアの国内品はNPN前提の設計が多い |

## NPN/PNP両対応ユニット 型番例

| メーカー | ユニット型番 | 切替方法 | 備考 |
|---------|------------|---------|------|
| 三菱MELSEC | QX40 / QX41 | COM端子の+24V or GND接続で切替 | Qシリーズ標準 |
| オムロン | CJ1W-ID231 | COM配線で切替 | NX/CJシリーズ |
| 富士電機 | NP1X16S | COM配線で切替 | NP1シリーズ |
| キーエンス | KV-8000シリーズ | ユニット設定で切替 | — |

!!! note "切替コストは配線変更のみ"
    両対応ユニットでNPN↔PNP切替はCOM端子の接続先を変えるだけ。
    片方専用ユニットを誤発注した場合はユニット交換（数万円）になる。

## 配線例

=== "NPN出力 ＋ ソース型DI入力（国内標準）"

    ```
    センサ        PLC DI入力（ソース型）
    +24V ──────── センサ電源(+)
    OUT  ────────┐
                 ├── DI端子（IN）
                 │   (内部: +24V → DI端子 → IN回路 → COM)
    GND  ──────── センサ電源(-)
                     COM（+24V接続 ＝ +COM）
    ```

    - DI入力の COM は +24V に接続（ソース型の特徴）
    - センサON時：OUT端子がGND電位になり電流が流れてON判定

=== "PNP出力 ＋ シンク型DI入力（欧州機器の場合）"

    ```
    センサ        PLC DI入力（シンク型）
    +24V ──────── センサ電源(+)
    OUT  ────────┐
                 ├── DI端子（IN）
                 │   (内部: COM → IN回路 → DI端子)
    GND  ──────── センサ電源(-)
                     COM（GND接続 ＝ -COM）
    ```

    - DI入力の COM は GND に接続（シンク型の特徴）
    - センサON時：OUT端子が+24V電位になり電流が流れてON判定

## NPN/PNP 混在時の対処

既設設備に別メーカーのセンサを追加する際、NPN/PNPが混在することがある。

### 方法1：リレーで変換（推奨）

```
NPN出力センサ → 中間リレー → PLC DI入力（任意タイプ）
```

- コスト: 1,500〜3,000円/点
- メリット: 絶縁できる、電圧変換も同時に可能
- デメリット: 盤スペースが必要

### 方法2：フォトカプラで変換

- コスト: 500〜1,500円/点
- メリット: 小型・省スペース
- デメリット: 設計工数がかかる（定格計算が必要）

### 方法3：NPN/PNP切替可能なPLC入力ユニット使用

三菱MELSEC・オムロンCJ/NX等の上位機種では、DI入力の**COM結線を変えるだけ**でNPN/PNP両対応になる機種がある。新規設計時に採用すると混在問題が起きない。

!!! tip "新規盤設計での推奨"
    DI入力ユニットは「NPN/PNP両対応タイプ」を標準化しておくと、将来的な機器追加・変更時の柔軟性が高い。

## センサ出力電流定格の確認

センサのNPN/PNP出力トランジスタには**定格電流**がある。DI入力の消費電流がこれを超えると出力素子が破損する。

| センサ種別 | 一般的な出力電流定格 | DI入力消費電流の目安 |
|----------|-----------------|----------------|
| 近接センサ（小型） | 100 mA | 5〜10 mA |
| 近接センサ（汎用） | 200 mA | 5〜10 mA |
| 光電センサ | 100〜200 mA | 5〜10 mA |
| リードスイッチ | — | — |

通常のPLC DI入力（5〜10 mA）であれば定格内。ただし以下のケースは要注意。

!!! warning "並列接続・複数DI入力への分岐"
    1つのセンサ出力を複数のDI入力に並列接続すると電流が加算される。
    DI入力2点並列（10mA×2=20mA）でも多くの場合問題ないが、定格を確認してから設計すること。

!!! warning "リレーコイルの直接駆動"
    センサ出力でリレーコイルを直接駆動する場合、コイル電流（50〜100mA）が定格に近い。
    サージ電圧によるトランジスタ破損防止のため**フライバックダイオードを必ず追加**する。

---

## センサの出力タイプ確認方法

1. **カタログの出力回路図を見る** — 「NPN」「PNP」「シンク」「ソース」の表記を確認
2. **型番から判断** — 末尾の英数字で区別する機種が多い（例：E2E-X5ME1 = NPN, E2E-X5ME2 = PNP）
3. **テスターで実測** — センサON時に出力端子とGND間の電圧を測定：0V近ければNPN、+24V近ければPNP

## よくある誤配線・トラブル

!!! failure "OUT端子が「常にON」または「反応しない」"
    NPN出力センサをシンク型DI入力（-COM）に接続した場合。電流が正しく流れず、常時ON/OFFになる。
    **→ PLCのCOM配線を+24Vに変更（ソース型に切替）するか、センサをPNP型に交換。**

!!! failure "センサの出力端子がすぐ故障する"
    出力端子を+24Vまたは0Vに直結してしまった場合（過電流）。センサ出力端子は必ずDI入力経由で使うこと。
    **→ DI入力なしにテスターで導通確認するのも危険。ループキャリブレータ等を使う。**

!!! failure "長距離配線でのON誤検知"
    PNPセンサで長い配線（数十m）を使うと、漏れ電流でDI入力がONと判定されることがある。
    **→ NPN出力センサに変更するか、漏れ電流補正機能付きのDI入力ユニットを使用。**

!!! warning "NAMUR出力は別物"
    欧州製の防爆センサでよく使われる「NAMUR出力」はNPN/PNPとは異なる（電流変化で伝達）。
    NAMUR対応のバリア・アンプが別途必要。混同しないよう注意。

## 実務チェックリスト

計器選定・工事前に以下を確認する。

- [ ] PLCのDI入力ユニット仕様書でNPN/PNP対応タイプを確認した
- [ ] PLCのCOM端子が+24V接続（ソース型）か GND接続（シンク型）かを確認した
- [ ] センサカタログで出力タイプ（NPN/PNP）を確認した
- [ ] 既設センサと追加センサで出力タイプが混在しないか確認した
- [ ] 混在する場合の変換方法（リレー等）を設計に含めた
- [ ] 防爆エリアのセンサはNAMUR出力の有無を確認した
- [ ] センサ出力電流定格 ≥ DI入力消費電流（並列接続時は合算）を確認した
- [ ] リレーコイル直接駆動の場合はフライバックダイオードを設計に含めた
- [ ] 仕様書に「DI入力：NPN/PNP両対応、DC24V」と明記した（新設PLCの場合）
