# 計装基礎

## 30秒まとめ

4-20mAは「電流ループ」で信号伝送する業界標準。4mAがゼロスケール（断線と区別できる）、20mAがフルスケール。電圧伝送より耐ノイズ性が高く、長距離伝送に向く。HARTはこのアナログ信号にデジタル通信を重畳する技術。

---

## 4-20mAループの仕組み

### なぜ4mAがゼロか

| 電流値 | 区分 | 意味 |
|--------|------|------|
| 0 mA | — | 断線・電源失 |
| ≤ 3.6 mA | 故障判定（NAMUR NE43） | 断線・センサ異常（フェイルロー方向） |
| 3.8 mA | 有効測定範囲 下限 | これ未満はアンダーレンジ |
| 4 mA | ゼロスケール | 正常な最小信号 |
| 20 mA | フルスケール | 正常な最大信号 |
| 20.5 mA | 有効測定範囲 上限 | これ超はオーバーレンジ |
| ≥ 21 mA | 故障判定（NAMUR NE43） | センサ異常（フェイルハイ方向） |

!!! note "「有効測定範囲」と「故障判定閾値」は別物"
    - **有効測定範囲 3.8〜20.5 mA**: メーカが直線性を保証する計測有効範囲。この外はアンダー／オーバーレンジ。
    - **故障判定 ≤3.6 mA ／ ≥21 mA（NAMUR NE43）**: 断線・センサ異常を検出するフェイル判定の閾値。
    - 3.6〜3.8 mA・20.5〜21 mA は遷移域（通常は使用しない領域）。
    数字を混同しやすいので、「計測の有効範囲」か「故障検出の閾値」かを区別して扱います。

0mAを「生きている信号」として使わないことで、**断線と測定値ゼロを区別できる**。これが4-20mAの最大の利点。現場でアラームが出たとき、まず「0mAか否か」を確認する習慣をつける。

### ループの電流計算

```
受信計器の表示値 = (I - 4mA) / 16mA × スパン + ゼロ点
```

例：0〜100℃ スパン、受信電流12mA の場合
```
(12 - 4) / 16 × 100 = 50℃
```

## 電流伝送 vs 電圧伝送

| 項目 | 電流伝送 (4-20mA) | 電圧伝送 (1-5V等) |
|------|------------------|------------------|
| 耐ノイズ性 | 高い | 低い |
| 長距離伝送 | 可（数百m） | 不向き（電圧降下） |
| 配線抵抗の影響 | 受けにくい | 受ける |
| 主な用途 | フィールド〜DCS間 | 盤内短距離 |

電流伝送が耐ノイズ性に優れる理由：ノイズは電圧として乗ってくるが、電流ループでは受信側の入力抵抗両端の電圧誤差にしかならず、電流自体は変化しない。

!!! warning "配線抵抗の上限"
    ループ電源電圧（通常24V）から伝送器の最低動作電圧（約12V）を引いた残りが許容電圧降下。
    許容負荷抵抗 = (24V - 12V) / 0.020A = **600Ω** が一般的な上限目安。
    受信計器の入力抵抗＋ケーブル抵抗の合計がこれを超えないこと。

## HART通信の概要

HART（Highway Addressable Remote Transducer）は、4-20mAアナログ信号に**1200bpsのデジタル信号をFSK変調で重畳**する通信方式。

```
アナログ信号  4mA ────────────────────── 20mA
デジタル信号  ~~~~波形（1200Hz/2200Hz）~~~~
             同一のケーブル上に共存
```

FSK（Frequency Shift Keying）で、論理「1」を1200Hz・論理「0」を2200Hzの正弦波に割り当て、平均値がゼロになる交流波（振幅 ±0.5 mA）としてアナログ電流に重畳します。平均電流が変化しないため、4-20mAのアナログ計測値を乱さずにデジタル通信を同時に流せます。

### HARTでできること

- 機器情報の読み出し（製造番号・タグ・ステータス）
- 設定変更（レンジ・単位・減衰）
- 診断情報の取得（自己診断結果）
- 複数パラメータの同時取得（最大4変数）

!!! tip "現場での活用"
    ループキャリブレータ（FLUKE 707 等）にHARTモデムを繋げると、フィールドで伝送器のゼロ・スパン調整や診断ができる。工事・保全作業の効率が大きく上がる。

### ループ抵抗の要件（重畳通信の成立条件）

HARTのFSK信号は電流変調です。マスタ（ハンドヘルドや変調器）が読み取るには、ループのどこかに抵抗があって電流変動を電圧信号に変換する必要があります。

- **標準 250 Ω**: 4mA→1V・20mA→5V に変換でき、DCSのアナログ入力の受信抵抗としても一般的。
- **下限 230 Ω**: これを下回るとFSK信号の振幅が不足し、通信が不安定になります。
- **上限の目安 約 600 Ω**: HART仕様が推奨する範囲は 230〜600 Ω。これ以上はループ電源電圧との兼ね合い（前述「電流伝送 vs 電圧伝送」の配線抵抗の上限で述べた電圧降下）で伝送器が動かなくなります。

!!! warning "HART通信できないときの定番原因"
    受信抵抗が 230 Ω 未満、あるいは通信機の接続位置に抵抗が挟まっていないケースが多いです。DCS入力抵抗が低い盤では、伝送器端子に外付け 250 Ω を入れて通信することがあります。診断手順は [信号系トラブル](../06-trouble/signal.md) を参照してください。

### マルチドロップ接続

1つのループに複数のHART機器（伝送器）をぶら下げる接続方式です。

- 各機器の**アナログ電流を 4 mA に固定**し、プロセス値はデジタル通信だけで読み出します（アナログ計測は使えません）。
- 機器はポーリングアドレスで識別します。**HART 5 まではアドレス 1〜15（最大15台）**、**HART 6/7 では拡張アドレスで最大63台**まで対応します。
- マスタが各機器を順番にポーリングするため、**更新周期が遅くなる**（1機器あたり数百ms〜秒オーダ）のがトレードオフです。応答速度が要る制御ループには不向きで、監視用の多点計測（タンクヤードの温度・液位など）に向きます。

### バーストモード

通常のHARTはマスタが要求してスレーブが応答する「要求応答」型ですが、バーストモードでは伝送器が要求を待たずに一定周期でデータを送出し続けます。ポーリングのオーバーヘッドがなくなり、更新周期を上げられます。WirelessHARTの定期送信もこの考え方を使います。

### WirelessHART（IEC 62591）

既設の4-20mA/HART配線を引き直さずに、無線でHARTデータを収集する方式です。

- 各フィールド機器が**自己組織化メッシュ**を形成し、機器同士が中継してゲートウェイまでデータを届けます。障害物や1ノード故障があっても経路を張り替えます。
- 既設の2線式伝送器には**WirelessHARTアダプタを後付け**でき、配線工事なしで診断情報を吸い上げられます。
- 用途は監視・診断・非制御の補助計測が中心で、応答性やSIL要求の高い制御ループには使いません。

### HART値の一括読み出し

多数のHART機器の診断情報を活用するには、盤側でデジタル値を取り込む仕組みが要ります。

- **HART対応I/Oカード**: DCS/PLCのアナログ入力カード自体がHARTを復調し、アナログ値とデジタル診断を同時に取り込みます。新設プラントの標準的な構成です。
- **HARTマルチプレクサ**: 既設の非HART入力カードを活かしたまま、端子台に多重化装置を追加してHART値を別系統（保全用アセットマネジメントなど）へ吸い上げます。既設改造で使われます。

## 2線式と4線式

### 2線式（ループパワード）

```
[DCS/バリア] ─── (+) ─── [伝送器] ─── (-) ─── [DCS/バリア]
              24VDCを供給         信号電流(4-20mA)を返す
```

- 電源と信号を**同一の2本のケーブル**で賄う
- 伝送器の消費電力は最大4mA×最低動作電圧で制限される
- 配線コストが安い → 現場では主流

### 4線式（セパレートパワード）

```
信号線  ─ (+) / (-) ─  伝送器
電源線  ─ (+) / (-) ─  伝送器（別電源 100/200VAC または 24VDC）
```

- 分析計・ガス検知器・超音波流量計など**消費電力の大きい機器**に使用
- 電源と信号が独立しているため、電源電圧の変動が信号に影響しない

## フィールドバスのセグメント設計

4-20mA/HARTが「1機器＝1ループ」なのに対し、**フィールドバス（Fieldbus）は1本のケーブル（セグメント）に複数のディジタル機器を接続**します。化学プラントで使う代表は **FOUNDATION Fieldbus H1** と **PROFIBUS PA** で、どちらも **IEC 61158-2 の同じ物理層（31.25 kbps・MBP：Manchester Bus Powered、2線でバス給電）** を共有します。規格の位置づけは [計装工事で押さえるJIS規格](jis-standards.md) の「B. 信号」を参照してください。

### トランクとスパーの構造

```text
[電源コンディショナ]─┬─ トランク（幹線）───────────────┬─[終端抵抗]
   [H1 I/Oカード]    │                                  │
                     ├─スパー─[機器]   ├─スパー─[機器] │
                     │        (支線)   │               │
                   [終端抵抗]        ジャンクションボックス
```

- **トランク（幹線）**: セグメントの背骨。両端に終端抵抗を1個ずつ置きます。
- **スパー（支線）**: トランクから各機器へ分岐する短い枝線。ジャンクションボックスやデバイスカプラで分けます。

### 終端抵抗は「両端に2個」

- セグメントには**終端抵抗をちょうど2個**（トランクの両端に1個ずつ）入れます。中身は 100 Ω 抵抗と 1 μF コンデンサの直列で、信号の反射を防ぎます。
- **1個不足**（1個だけ）だと反射で波形が乱れ通信エラー、**3個以上**（過剰）だと信号振幅が下がって同じく通信不良になります。増設・改造で電源コンディショナ内蔵の終端を二重に入れてしまう事故が定番です。

### 電力バジェット（給電設計の要）

バス給電（2線で電源と通信を兼ねる）のため、**全機器の消費電流とケーブル電圧降下を積み上げ、セグメント末端の機器電圧が最低動作電圧を割らない**ことを設計時に確認します。

- FF H1 機器の**最低動作電圧は 9 V DC**。電源コンディショナ出力（トランク始点で約 24〜32 V）から、全機器の消費電流 × ケーブル抵抗で生じる電圧降下を引いた末端電圧が 9 V 以上必要です。
- 機器1台の消費電流は**おおむね 10〜20 mA**（機器仕様値）。設計では合計電流に**約10%の余裕**を見込みます。
- 冗長化のため電源コンディショナは二重化して設置するのが一般的です。

### セグメント長・スパー長・機器台数の制約

| 項目 | 目安（Type A ケーブル・非本安） | 備考 |
|------|------------------------------|------|
| バス速度 | 31.25 kbps | FF H1／PROFIBUS PA 共通 |
| トランク最大長 | 1900 m | ケーブル種別で変わる |
| スパー最大長 | 60 m | 台数が増えるほど短く制限 |
| 1セグメント機器台数 | 4〜16台程度 | 電力バジェットと実行周期で決まる。非本安で最大16台程度 |
| 終端抵抗 | 2個（両端） | 過不足厳禁 |

!!! note "台数は「電力」と「速度」の二重制約"
    電力バジェット上は載っても、機器が増えるとマクロサイクル（全機器を巡回する制御周期）が延びます。速い制御が要るループは台数を絞るか、4-20mAを併用します。

### 本安（防爆）との関係 — FISCO/FNICO

危険区域でフィールドバスを本質安全（Ex i）にする設計コンセプトが **FISCO（Fieldbus Intrinsically Safe Concept）** と **FNICO（非点火：Zone 2向け）** です。

- FISCOでは機器・ケーブル・電源のエンティティパラメータを規格枠内に収めることで、個別の本安計算を簡略化します。
- 本安にすると給電電力が絞られるため**台数と距離はさらに制限**されます（例: IIC で幹線約 1 km・数台程度、IIB/IIA で最大 1.9 km）。スパー長は最大 60 m 目安です。
- 本安の基本概念・エンティティパラメータは [防爆](explosion-proof.md) を参照してください。

## 4-20mA / HART / フィールドバスの使い分け

| 観点 | 4-20mA（アナログ） | HART（重畳） | フィールドバス（H1/PA） |
|------|------------------|-------------|----------------------|
| 配線 | 1機器1ループ | 1機器1ループ（既設流用可） | 1セグメントに複数台 |
| 診断情報 | なし（電流のみ） | あり（アナログ優先＋診断） | 豊富（全ディジタル） |
| 制御応答 | 速い | 速い（アナログは即時） | セグメント台数依存でやや遅い |
| 配線量・コスト | 点数比例で増える | 4-20mAと同等 | ケーブル本数を大きく削減 |
| 既設流用 | — | そのまま診断追加可（後付けHART/無線） | 基本は新設・専用設計 |
| 向く用途 | 単純ループ・安全計装 | 既設の診断強化・多点監視 | 新設大規模プラントの機器集約 |

- **既設プラントの診断強化**なら、配線を変えずに済むHART（対応I/O・マルチプレクサ・WirelessHART）が第一候補です。
- **新設で機器点数が多い**なら、ケーブルを削減できるフィールドバスが効きます。ただし終端・電力・台数のセグメント設計が前提です。
- **安全計装（SIS）や高速・単純なループ**は、シンプルで独立性の高い4-20mAを選ぶのが基本です（SIL設計は [安全計装 SIS/SIL](sis-sil.md) 参照）。

## よくある現場トラブル

!!! failure "断線との区別"
    DCSのトレンドで突然0%（4mA相当）になったとき、プロセス値が本当にゼロになったのか断線かを確認する。ループチェッカーで実電流を測定する。

!!! failure "アース（シールド）問題"
    シールドの多点接地によるグラウンドループで、トレンドにノイズが乗ることがある。シールドは片端接地（DCS側）が原則。

!!! failure "負荷抵抗オーバー"
    安全バリア（本安バリア）を後付けする際、ループ抵抗が増えて伝送器が正常動作しないことがある。増設前にループ抵抗計算を確認する。
