# インバータトラブル

!!! danger "⚡ まず確認（3秒）"
    1. **エラーコード**を表示パネルで確認・メモする
    2. **エラー発生時刻**をDCS趨勢で確認する
    3. **周囲温度・冷却ファン**を目視確認する

    🔄 リセット前に必ず原因を特定すること。繰り返しトリップは機器損傷のサイン。

## 30秒まとめ

インバータトラブルはエラーコードから「過電流・過電圧・過熱・地絡」の4系統に分類して切り分ける。ノイズ対策と通信配線の確認が8割の問題を解決する。

---

## 主要エラーコード一覧

※ エラーコードはメーカー・機種により異なる。以下は代表的な汎用記号。実機の取扱説明書と照合すること。

| コード | 意味 | 主な原因 | 優先確認事項 |
|--------|------|---------|------------|
| OC（E01） | 過電流 | 加速時：加速時間が短すぎる / 定常：過負荷・軸受ロック | 加速時間・負荷確認 |
| OV（E03） | 過電圧 | 減速時：減速時間が短すぎる / 電源電圧が高い | 減速時間・電源電圧 |
| UV（E04） | 不足電圧 | 電源電圧低下・瞬停・入力ヒューズ断 | 電源電圧・ヒューズ |
| OH（E05） | 過熱 | 周囲温度高・冷却ファン故障・フィルタ詰まり | 周囲温度・ファン・フィルタ |
| GF（E06） | 地絡 | モーター巻線の対地絶縁低下・ケーブル損傷 | 絶縁抵抗測定 |
| SC | 短絡 | 出力端子間短絡・モーター内部短絡 | 配線・モーター確認 |
| CPU | CPU 異常 | ノイズ・電源瞬停・内部故障 | 再起動・ノイズ対策 |
| LF | 欠相 | 入力または出力の欠相 | 配線・ヒューズ確認 |
| COM | 通信異常 | RS-485 配線断・設定ミス・ノイズ | 配線・設定確認 |

!!! tip "エラー発生直前の DCS 趨勢を確認"
    エラーが発生した時刻前後の電流・速度・温度の趨勢を DCS で確認すると原因特定が速い。
    「急激な電流上昇 → OC」「減速中の電圧上昇 → OV」のように発生パターンで絞り込める。

## 過電流（OC）の切り分け

発生タイミングによって原因が異なる。

=== "加速中に OC"
    **原因：** 加速時間が短すぎる / 負荷イナーシャが大きい

    **対処：**
    1. 加速時間パラメータを延長する（例：5秒 → 10秒）
    2. トルクブースト設定の見直し
    3. 負荷が重すぎる場合はモーター容量の見直しを検討

=== "起動直後（低速時）に OC"
    **原因：** トルク不足 / V/f 設定不適切 / 負荷ロック

    **対処：**
    1. 低速トルクブーストを上げる
    2. 負荷を切り離して空転確認（空転でも OC なら機器内部の問題）
    3. センサレスベクトル制御に変更を検討

=== "定常運転中に OC"
    **原因：** 過負荷・軸受ロック・相欠け

    **対処：**
    1. クランプメーターで出力3相電流を測定
    2. 電流が定格を超えていたら負荷側を確認
    3. 軸受温度・振動を確認

## ノイズ起因の誤動作

インバータはスイッチング動作によりノイズを発生する。これが同じ盤内・同じルートを通る計装信号に影響することがある。

### ノイズ経路と対策

| ノイズ経路 | 症状 | 対策 |
|----------|------|------|
| 電源ラインノイズ | 同一電源の計器が誤動作 | ラインフィルタ追加・電源分離 |
| 放射ノイズ | 近傍の計装ケーブルが影響 | 計装ケーブルとの離隔確保・シールド |
| グラウンドループ | 信号がドリフト・ノイズ重畳 | シールド片端接地・グランド分離 |

### チェックポイント

- [ ] 計装ケーブルのシールドが**片端接地**になっているか（両端接地はグラウンドループの原因）
- [ ] インバータ出力ケーブルと計装ケーブルが**同一配管・同一ダクト**を通っていないか
- [ ] インバータのアース（PE 端子）が**専用接地線で確実に接地**されているか
- [ ] インバータ出力側に**ラインフィルタ・EMC フィルタ**が設置されているか

!!! warning "インバータ近傍の計器誤動作"
    インバータを設置または増設した後から計器の値が暴れるようになった場合、
    ノイズの影響が最有力候補。配線ルートの見直しと EMC 対策を優先する。

## 通信異常の確認順序

DCS/PLC とインバータ間の通信（RS-485 / Modbus / Profibus 等）が切れた場合：

1. **物理的な接続確認**
   - 通信ケーブルの断線・コネクタ抜け・端子の緩み
   - ターミネーション抵抗（終端抵抗）の設置状態（RS-485 は両端に 120Ω 必要）

2. **設定確認**
   - インバータ側の通信設定（ボーレート・パリティ・局番）が DCS 側と一致しているか
   - 制御モードが「通信制御」になっているか

3. **ノイズ確認**
   - 通信ケーブルが動力ケーブルと並走していないか
   - シールドが適切に接地されているか

4. **ハードウェア確認**
   - 通信カード・オプションカードの接触不良
   - 電源電圧の安定性

!!! tip "通信断時のフォールバック設定"
    インバータに「通信断時の動作設定」がある。
    「通信断時に停止」「通信断時に前回値で継続」など設定により動作が変わる。
    プロセス安全上どちらが適切かを製造担当と事前に決めておくこと。

## パラメータバックアップの重要性

!!! danger "インバータ交換前にパラメータバックアップを必ず取る"
    インバータを交換すると、設定パラメータはすべて工場出荷値に戻る。
    バックアップなしで交換すると再設定に数時間かかることがある。

### バックアップ方法

| 方法 | 特徴 |
|------|------|
| コピーユニット（オプション品） | 本体から本体へパラメータをコピーできる。最も確実 |
| PC ソフト（メーカー提供） | USB/RS-485 経由でパソコンに保存。データ管理に適している |
| 手書き記録 | コピーユニットが使えない場合の最終手段。時間がかかる |

バックアップは**定修の都度**取得し、変更した場合は都度更新する。
保管場所：盤扉内の資料ポケット + 電子データ（サーバー）の2か所に保存する。

## 防爆エリアのインバータ設置

!!! danger "防爆エリアにインバータを直接設置してはいけない"
    インバータは防爆構造ではない（通常品）。
    防爆エリアに設置する場合は**盤内設置（安全場所への盤設置 + 長距離配線）**が原則。

### 長距離配線時の注意（サージ電圧問題）

インバータ〜モーター間のケーブルが長い場合、単なるノイズではなく **モーター端のサージ電圧（マイクロサージ）** がモーター絶縁と軸受を痛める。防爆エリア対応で盤を安全場所に置くと配線が長くなりやすく、この問題が顕在化する。

#### なぜ起きるのか（現象の機序）

- PWM インバータの出力は急峻な立ち上がり（dv/dt が大きい方形波パルス）を持つ
- このパルスがケーブルを **進行波** として伝わり、インバータ側とモーター側の **サージインピーダンス不整合** によりモーター端で反射する
- 反射波が入射波に重畳し、モーター端子電圧が最大で **直流中間電圧の約 2 倍（対地サージ電圧）** まで跳ね上がる（例：400 V 級で 1,000〜1,400 V 級のピークが実測される）
- ケーブル長が **クリティカル長**（パルス立ち上がり時間とケーブルの波動伝搬速度で決まる。400〜480 V 級でおおむね 30〜50 m 目安）を超えると重畳が顕著になり、長いほど悪化する
- 繰り返すサージがモーター巻線の絶縁を劣化させ（コロナ放電・部分放電）、さらに高周波電流が軸受を通って流れると **軸受電食（電蝕）** を招く

!!! warning "長距離配線＝ノイズではなくサージ絶縁問題"
    盤内 EMC 対策（フィルタ・接地）で計器ノイズは収まっても、モーター端のサージ電圧は別問題。モーターの巻線焼損・軸受の早期損傷が繰り返す場合はサージを疑う。

#### 切り分け（現場で打てる手）

- [ ] **配線長とケーブル種別を確認**：実配線長（往復ではなく片道の布設長）と、シールド付き／3芯一括か単芯かを図面・現物で確認する
- [ ] **発生タイミングを切り分け**：巻線焼損・地絡（GF）の再発が「高速運転時」「起動直後」のどちらに偏るかを DCS 趨勢で確認する（高速・高キャリアほどサージ頻度が上がる）
- [ ] **モーター端電圧を観測**（可能な場合）：モーター端子箱でオシロ＋高圧差動プローブにて対地ピーク電圧・立ち上がり時間を測定する。定格の 2 倍近いピークが出ていればサージ確定
    - オシロ観測が難しい現場では、まず後述の応急（キャリア低減）で症状が軽くなるかで間接的に判断する
- [ ] **応急でキャリア周波数を下げる**：キャリア周波数を下げるとサージの繰り返し頻度が減り、絶縁ストレスを緩和できる（恒久策ではないが被害進行を抑える応急。ただし低速トルク・騒音は要確認）

!!! danger "サージ観測は活線高電圧作業"
    モーター端でのオシロ観測は通電中の高電圧測定になる。高圧差動プローブと保護具を用い、感電・短絡に十分注意する。無理な場合は測定を依頼判断に含める。

#### 対策の選択肢と使い分け

ケーブル長・モーターの耐サージ性能に応じて、dv/dt を鈍らせる側とモーター端で受ける側を組み合わせる。

| 対策 | 設置位置 | 効果 | 主な適用距離の目安 |
|------|---------|------|-----------------|
| キャリア周波数低減 | インバータ設定 | サージ繰り返し頻度を下げる応急・補助 | 全距離（単独では不十分） |
| 出力リアクトル（AC リアクトル／dv/dt リアクトル） | インバータ二次側 | dv/dt を鈍らせ、浮遊容量充電電流とサージを低減 | 50 m 超の一次対策 |
| dv/dt フィルタ | インバータ二次側 | 立ち上がりをさらに緩やかにしピーク電圧を抑制 | 100 m 超 |
| 正弦波フィルタ（sine wave filter） | インバータ二次側 | 出力をほぼ正弦波化しサージをほぼ解消 | 長距離・耐サージ性の低いモーター／複数台まとめて |
| サージ電圧抑制ユニット（ターミネータ） | モーター端子箱側 | 反射をモーター端で吸収しピークをクランプ | オシロで端子電圧上昇を確認した場合／既設補強 |

- **距離が主因**なら出力リアクトルを一次対策とし、足りなければ dv/dt フィルタ／正弦波フィルタへ段階的に強化する
- **モーター側が弱い**（汎用モーターで耐サージ設計でない）場合は正弦波フィルタかモーター端抑制ユニットを優先する
- リアクトル・フィルタは電圧降下（トルク低下）を伴うため、容量・インピーダンスはモーター定格と配線長に合わせて選定する

!!! tip "既設の 200m 超配線への追補"
    技術リファレンス（[低圧設備／インバータ](../02-teiatsu/inverter.md)）の通り、200 m を超えると出力リアクトル単独では不足しやすい。まず端子電圧を観測し、リアクトル＋dv/dt フィルタで足りなければ正弦波フィルタへ、という順で強化する。

#### 予防設計（新設・更新時）

- **配線長を短くする配置**：可能な限りインバータ盤をモーターに近づける。防爆エリア制約で離す場合は最初からリアクトル／フィルタを盤設計に織り込む
- **インバータ対応（絶縁強化）モーターを採用**：インバータ駆動を前提とした耐サージ絶縁のモーターを選ぶ。防爆エリアでは「インバータ対応防爆モーター」を指定する
- **軸受電食対策**：絶縁軸受（反負荷側）・シャフト接地ブラシ・軸電圧を逃がす接地の採用を検討する（モーター側の詳細は[モーター制御](../02-teiatsu/motor-control.md)を参照）
- **ケーブルはインバータ用シールドケーブルを選定**し、シールドは盤側で確実に接地する

#### メーカーへ相談する場合の判断基準

上記の切り分け・応急まで済ませた上で、次に該当すれば製作元・モーターメーカーへ相談する。

- モーター端電圧の実測ピーク・立ち上がり時間（観測できた場合の数値）
- 配線長・ケーブル種別・キャリア周波数の現状値
- 出力リアクトル装着後もサージ・絶縁劣化が再発する（フィルタ種別の選定・容量整合が必要）
- 該当モーターの耐サージ電圧（絶縁クラス）が不明で、インバータ対応モーターへの更新可否を確認したい

依頼時は「症状・現状値・実施済み対策」をセットで提示すると、フィルタ選定や更新提案が速い。

### 防爆モーターとインバータの組み合わせ

防爆エリアのモーターをインバータ制御にする場合：
- **インバータ対応防爆モーター**を使用すること（通常の防爆モーターはインバータ駆動で過熱する可能性がある）
- **自冷式モーターは低速時に冷却不足になる**ため、強制冷却ファン付きに変更するか、速度範囲を制限する

## インバータトラブル原因早見表

| 症状 | 最有力原因 | すぐやること |
|------|----------|-----------|
| OC が加速中に発生 | 加速時間が短い | 加速時間パラメータを延長 |
| OC が定常運転中に発生 | 過負荷 | 出力電流をクランプメーターで測定 |
| OV が減速中に発生 | 減速時間が短い | 減速時間パラメータを延長 |
| OH が頻発する | 周囲温度高・フィルタ詰まり | 周囲温度測定・フィルタ清掃 |
| 計器がノイズで暴れる | インバータノイズ | 配線ルート確認・EMC 対策 |
| 通信が頻繁に切れる | ノイズ・ターミネーション抵抗 | 通信ケーブル確認・終端抵抗確認 |
